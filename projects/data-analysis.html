Vertical Farm Telemetry Analysis

A pure Python data analysis system for a vertical farm. It reads a real telemetry ledger with 600 production rows and turns it into a clean dataset, analysis tables, charts, and a delivery-ready PDF report.

What it does

One pipeline converts a raw farm CSV into an organized dataset, analysis outputs, charts, and a portfolio-ready PDF — run end-to-end with a single command.

Pipeline steps

  • Import & clean (prepare_dataset.py) — reads vertical_farm_biomass_telemetry_ledger.csv, cleans columns, parses Stack / Rack / Row IDs, outputs an analysis-ready dataset
  • Analysis (analysis.py) — nutrient decay acceleration matrix for 60 irrigation channels, weak-circulation channels, harvest-weight index per crop, best nutrition balance from top yield rows
  • Charts (charts.py) — EC distribution, circulation vs nutrient decay, matrix heatmap, harvest index plot
  • Report (report.py) — 3-page PDF (Abd Alrahman Ahmed) with summary, matrix, and recommendations
  • Full run (run_pipeline.py) — executes every step with one command

Language & libraries

The entire project is written in Python only.

  • Python
  • pandas — read, clean, aggregate
  • numpy — calculations & normalization
  • matplotlib — charts + PDF
  • openpyxl — Excel export

Outputs

  • Clean structured dataset from raw telemetry CSV
  • Analysis tables (60 irrigation channels)
  • Charts: EC, circulation, heatmap, harvest index
  • Excel exports via openpyxl
  • 3-page PDF report ready for delivery / portfolio
